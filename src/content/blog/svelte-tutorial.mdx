## SSG vs SSR

- SSG: 当你运行 `npm run build` 这样的构建命令时，框架会预先为你的每一个页面生成一个独立的、完整的 HTML 文件。这些文件是静态的，不包含任何动态逻辑
- SSR: 服务器在接收到用户的页面请求后，会实时地获取最新的数据，渲染出完整的 HTML 页面，然后返回给浏览器。

## 元素InView的时候才进行动画效果

Svelte内置的 `transition:fly` 必须配置 `{#if}` 来实现动画效果。也就是当元素满足某种条件的时候，才进行动画效果。

最开始通过 `IntersectionObserver` 来实现 InView 的状态。这样元素出现在视图中的时候，就能有动画效果了。

但是当多个InView动画元素堆叠在一起的时候，比如下面这样

```Svelte
<div>
  {#if isInView}
    <div transition:fly={{y: 20, duration: 500}}>InView 元素1</div>
  {/if}
</div>
<div>
  {#if isInView}
	  <div transition:fly={{y: 20, duration: 500}}>InView 元素1</div>
  {/if}
</div>
```

虽然说这样有了动画效果，但是由于isInView为false的时候，元素没有被渲染, 外层元素的高度为0，导致后面不该出现在视图中的元素出现在视图中了，提前进行了动画效果。这个是我不想要的。

我看到 Action 和 Attachment, 可以实现 InView 的效:

```Svelte
export const inViewFly = (options?: InViewFlyOptions): Attachment<HTMLDivElement> => {
  const {
    y = 20,
    x = 0,
    scale = 1,
    rotate = 0,
    duration = 500,
    easing = "ease-out",
    delay = 0,
  } = options || {};

  let hasIntersected = $state(false); // 记录是否已经交叉过

  return (element) => {
    const observer = new IntersectionObserver(([entry]) => {
      // 一旦进入视口，就标记为已交叉
      if (entry.isIntersecting) {
        hasIntersected = true;
      }
    }, options);

    observer.observe(element);

    element.style.opacity = hasIntersected ? "1" : "0"
    element.style.transform = hasIntersected ? "translate(0, 0) scale(1) rotate(0deg)" : `translate(${x}px, ${y}px) scale(${scale}) rotate(${rotate}deg)`
    element.style.transition = `all ${duration}ms ${easing} ${delay}ms`;

    return () => {
      observer.disconnect();
    }
  }
}
```

## 全局State

```ts
// @ts-nocheck
import { browser } from '$app/environment';

const STORAGE_KEY = 'sidebar-collapsed';

// 从 localStorage 读取初始值
let initialState = false;
if (browser) {
  const stored = localStorage.getItem(STORAGE_KEY);
  if (stored !== null) {
    initialState = stored === 'true';
  }
}

// 内部状态（不直接导出）
let collapsed = $state(initialState);

// 监听状态变化，自动保存到 localStorage
if (browser) {
  $effect(() => {
    localStorage.setItem(STORAGE_KEY, String(collapsed));
  });
}

// 导出 getter 函数（响应式）
export function getCollapsed() {
  return collapsed;
}

// 导出 setter 函数
export function setCollapsed(value: boolean) {
  collapsed = value;
}

// 导出 toggle 函数
export function toggleSidebar() {
  collapsed = !collapsed;
}
```

模式 3：工厂函数（创建独立实例）                                               
                                                                                
// counter.svelte.ts                                                           
export function createCounter() {                                              
  let count = $state(0)                                                        
  let double = $derived(count * 2)  // 派生状态                                
                                                                                
  return {                                                                     
    get count() { return count },                                              
    set count(value) { count = value },                                        
    get double() { return double },                                            
    increment() { count++ }                                                    
  }                                                                            
}                                                                              
                                                                                
// 使用（每次调用创建新实例）                                                  
import { createCounter } from '$lib/counter'                                   
const counter1 = createCounter()  // 独立实例                                  
const counter2 = createCounter()  // 独立实例                                  
                                                                                
优点：                                                                         
- ✅ 每个调用有独立状态                                                        
- ✅ 支持多个实例                                                              
- ✅ 可以使用 $derived

export class Counter {

总结：Svelte 5 的 composable 模式非常灵活，关键是根据场景选择： 

- 全局配置 → $state 对象                                                       
- 独立实例 → 工厂函数 + getter/setter                                          
- 服务端数据 → TanStack Query（你的情况）

## Svelte 5 响应式追踪断掉问题

问题背景                                                                       
                                                                                 
  在 createUserSettings 中，通过 getter 返回 userSettingsQuery.data              
  导致响应式追踪失效。                                                           
                                                                                 
  // ❌ 问题代码                                                                 
  return {                                                                       
    get settings() {                                                             
      return userSettingsQuery.data;  // 返回内部普通对象                        
    }                                                                            
  }                                                                              
                                                                                 
  // 组件中使用                                                                  
  const { settings } = createUserSettings();                                     
  const isConfigured = $derived(                                                 
    !!(settings?.ai?.current_stt_provider && settings?.ai?.current_stt_model)    
  );                                                                             
  // settings 值不会响应式更新                                                   
                                                                                 
  核心原理                                                                       
                                                                                 
  Svelte 5 响应式追踪机制                                                        
                                                                                 
  - 基于 Proxy 拦截：追踪属性访问路径                                            
  - 需要完整的代理链：每个属性访问都要能被拦截                                   
  - 关键：不能断掉响应式链路                                                     
                                                                                 
  // ✅ 正常追踪                                                                 
  $state(obj) ──▶ Proxy(level1) ──▶ Proxy(level2) ──▶ 属性访问                   
                                                                                 
  // ❌ 追踪断掉                                                                 
  $state(obj) ──▶ getter() ──▶ 普通对象 ──▶ 无法追踪                             
                                                                                 
  响应式追踪断掉的常见场景                                                       
                                                                                 
  1. 通过 getter 返回内部对象 ❌                                                 
                                                                                 
  let user = $state({                                                            
    address: { city: 'Tokyo' }                                                   
  });                                                                            
                                                                                 
  // ❌ 断掉                                                                     
  function getAddress() {                                                        
    return user.address;  // 返回普通对象                                        
  }                                                                              
  let city = $derived(getAddress().city);                                        
                                                                                 
  // ✅ 正确                                                                     
  let city = $derived(user.address.city);                                        
                                                                                 
  2. 解构赋值 ❌                                                                 
                                                                                 
  let user = $state({ name: 'John', age: 30 });                                  
                                                                                 
  // ❌ 断掉                                                                     
  let { name } = user;                                                           
                                                                                 
  // ✅ 正确                                                                     
  let name = $derived(user.name);                                                
                                                                                 
  3. 保存外部引用 ❌                                                             
                                                                                 
  let user = $state({ hobbies: ['reading'] });                                   
                                                                                 
  // ❌ 断掉                                                                     
  let myHobbies = user.hobbies;                                                  
                                                                                 
  // ✅ 正确                                                                     
  let myHobbies = $derived(user.hobbies);                                        
                                                                                 
  4. 返回 query.data（内部值）❌                                                 
                                                                                 
  // ❌ 断掉：返回内部普通对象                                                   
  get settings() {                                                               
    return userSettingsQuery.data;                                               
  }                                                                              
  // 使用：settings?.ai?.current_stt_provider                                    
                                                                                 
  5. 返回整个 query 对象 ✅                                                      
                                                                                 
  // ✅ 正常：返回响应式对象                                                     
  get query() {                                                                  
    return userSettingsQuery;                                                    
  }                                                                              
  // 使用：query.data?.ai?.current_stt_provider                                  
                                                                                 
  原理图解                                                                       
                                                                                 
  正常追踪链路                                                                   
                                                                                 
  ┌─────────┐     ┌─────────┐     ┌─────────┐                                    
  │ $state  │ ──▶ │ Proxy   │ ──▶ │ Proxy   │                                    
  │ source  │     │ level 1 │     │ level 2 │                                    
  └─────────┘     └─────────┘     └─────────┘                                    
       ▲              ▲              ▲                                           
       └──────────────┴──────────────┘                                           
        Svelte 可以追踪整个路径                                                  
                                                                                 
  断掉追踪链路                                                                   
                                                                                 
  ┌─────────┐     ┌─────────┐     ┌─────────┐                                    
  │ $state  │ ──▶ │ getter   │ ──▶ │ 普通对象 │                                  
  │ source  │     │ 函数     │     │ (无代理) │                                  
  └─────────┘     └─────────┘     └─────────┘                                    
       ▲                              ▲                                          
       └──────────────────────────────┘                                          
        ❌ 链条在这里断了！                                                      
                                                                                 
  你的代码问题可视化                                                             
                                                                                 
  // 当前代码（断掉）                                                            
  createQuery() ──▶ query.data ──▶ get settings() ──▶ 普通对象 ❌                
                                                ▲                                
                                      追踪在这里断掉                             
                                                                                 
  // 应该改成（正常）                                                            
  createQuery() ──▶ query.data ✅                                                
                          ▲                                                      
                   直接访问，保持追踪                                            
                                                                                 
  解决方案对比                                                                   
  ┌────────────┬──────────────────────────────────────┬──────────┬────────────┐  
  │    方案    │                 代码                 │ 是否断掉 │   推荐度   │  
  ├────────────┼──────────────────────────────────────┼──────────┼────────────┤  
  │ 返回 data  │ get settings() { return query.data } │ ❌ 是    │ ⭐         │  
  ├────────────┼──────────────────────────────────────┼──────────┼────────────┤  
  │ 返回 query │ get query() { return query }         │ ✅ 否    │ ⭐⭐⭐     │  
  ├────────────┼──────────────────────────────────────┼──────────┼────────────┤  
  │ 直接暴露   │ query: userSettingsQuery             │ ✅ 否    │ ⭐⭐⭐⭐⭐ │  
  └────────────┴──────────────────────────────────────┴──────────┴────────────┘  
  最佳实践代码                                                                   
                                                                                 
  推荐方案：直接暴露 query                                                       
                                                                                 
  // createUserSettings.svelte.ts                                                
  export function createUserSettings() {                                         
    const userSettingsQuery = createQuery(() => ({                               
      queryKey: ['settings', 'user'],                                            
      queryFn: async (): Promise<UserSettings> => {                              
        const result = await settingsCommands.getUserSettings();                 
        if (result.status === 'error') {                                         
          throw new Error(result.error);                                         
        }                                                                        
        return result.data;                                                      
      },                                                                         
      staleTime: 1000 * 60 * 5,                                                  
    }));                                                                         
                                                                                 
    const updateUserSetting = createMutation(() => ({                            
      mutationFn: async ({ path, value }: { path: string; value: JsonValue }) => 
  {                                                                              
        const result = await settingsCommands.updateUserSetting(path, value);    
        if (result.status === 'error') {                                         
          throw new Error(result.error);                                         
        }                                                                        
        return result.data;                                                      
      },                                                                         
      onSuccess: () => {                                                         
        queryClient.invalidateQueries({ queryKey: ['settings', 'user'] });       
      },                                                                         
    }));                                                                         
                                                                                 
    return {                                                                     
      // ✅ 直接暴露 query 对象                                                  
      query: userSettingsQuery,                                                  
                                                                                 
      updateUserSetting: (path: string, value: JsonValue) => {                   
        updateUserSetting.mutate({ path, value });                               
      },                                                                         
    };                                                                           
  }                                                                              
                                                                                 
  组件中使用                                                                     
                                                                                 
  <script>                                                                       
    const { query, updateUserSetting } = createUserSettings();                   
                                                                                 
    // ✅ 直接访问 query.data，响应式正常工作                                    
    const isConfigured = $derived(                                               
      !!(                                                                        
        query.data?.ai?.current_stt_provider &&                                  
        query.data?.ai?.current_stt_model                                        
      )                                                                          
    );                                                                           
                                                                                 
    const handleSelectProvider = (provider: string) => {                         
      updateUserSetting("ai.current_stt_provider", provider);                    
    };                                                                           
  </script>                                                                      
                                                                                 
  <div>                                                                          
    {query.data?.ai?.current_stt_provider}                                       
    <!-- 响应式更新正常 -->                                                      
  </div>                                                                         
                                                                                 
  快速测试方法                                                                   
                                                                                 
  <script>                                                                       
    let count = $state(0);                                                       
                                                                                 
    // ❌ 断掉响应式                                                             
    let getCount = () => count;                                                  
    let double1 = getCount() * 2;  // 不会更新                                   
                                                                                 
    // ✅ 正常响应式                                                             
    let double2 = $derived(count * 2);  // 会更新                                
  </script>                                                                      
                                                                                 
  <button onclick={() => count++}>                                               
    Count: {count}                                                               
  </button>                                                                      
                                                                                 
  <p>Double 1 (断掉): {double1}</p>                                              
  <p>Double 2 (正常): {double2}</p>                                              
                                                                                 
  关键要点总结                                                                   
                                                                                 
  1. 保持代理链完整：不要在中间插入返回普通对象的函数                            
  2. 避免解构：let { name } = state 会断掉响应式                                 
  3. 避免外部引用：let ref = state.obj 不会响应式更新                            
  4. 直接访问：使用 $derived(state.obj.prop) 而不是中间函数                      
  5. 返回响应式对象：如果必须通过函数返回，返回整个响应式对象而不是内部值        
                                                                                 
  TanStack Query v6 + Svelte 5 注意事项                                          
                                                                                 
  - Query 对象本身是响应式的                                                     
  - 直接访问 query.data、query.isLoading 等属性                                  
  - 不要通过 getter 包装 query.data                                              
  - 直接暴露 query 对象，保持响应式链路完整
