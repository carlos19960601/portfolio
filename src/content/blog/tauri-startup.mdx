## React版本

1. 初始化项目

```shell
cargo create-tauri-app
```

2. 安装shadcn

```shell
bun add tailwindcss @tailwindcss/vite
```

App.css 文件中添加

```css
@import 'tailwindcss';
```

tsconfig.json 文件中添加

```json
{
  "compilerOptions": {
    "baseUrl": ".",
    "paths": {
      "@/*": ["./src/*"]
    }
  }
}
```

```shell
bun add -D @types/node
```

vite.config.ts 文件中添加

```ts
import path from 'path'
import tailwindcss from '@tailwindcss/vite'

export default defineConfig({
  plugins: [react(), tailwindcss()],
  resolve: {
    alias: {
      '@': path.resolve(__dirname, './src'),
    },
  },
})
```

```shell
bunx --bun shadcn@latest init
```

```shell
bunx --bun shadcn@latest add button
```

## Svelte 版本

1. 初始化项目

```shell
cargo create-tauri-app
```

2. 添加tailwindcss

```shell
bun x sv add tailwindcss
```

3. 添加路径别名

tsconfig.json
```json
"compilerOptions": {
  "paths": {
     "$lib": ["./src/lib"],
     "$lib/*": ["./src/lib/*"]
   }
}
```

vite.config.js
```ts
export default defineConfig(async () => ({
  resolve: {
    alias: {
      $lib: path.resolve("./src/lib"),
    },
  },
})
```

4. 添加shadcn

```shell
bun x shadcn-svelte@latest init
```

## 常见操作

### 配置相关

#### TitleBarStyle

这个设置主要用于控制窗口标题栏的外观和行为，特别是在 macOS 上，因为它对系统原生外观的集成最为深入。在 Windows 和 Linux 上，这些选项的效果可能不那么明显或有不同。

`TitleBarStyle` 定义了应用程序窗口的标题栏区域（通常包含关闭、最小化、最大化按钮和标题文字）如何与你的 Web 内容（HTML）进行交互和显示。

**TitleBarStyle::Visible**

行为：

- 窗口有一个不透明的、由操作系统绘制的原生标题栏。
- 你的 Web 内容被限制在标题栏以下的内容区域内，无法绘制到标题栏上。
- 用户可以通过拖拽这个原生的标题栏来移动窗口。
- 窗口的控制按钮（交通灯按钮 on macOS）是标准的系统样式。

视觉效果：

```
+——————————————————————[原生标题栏]——————————————————————+
|  [绿灯] [黄灯] [红灯]            My App Title             |
+—————————————————————————————————————————————————————————+
|                                                         |
|                你的 Web 应用内容区域                    |
|                                                         |
|                                                         |
+—————————————————————————————————————————————————————————+
```

**TitleBarStyle::Transparent**

这个选项为你完全自定义标题栏打开了大门。

行为：

- 操作系统不再绘制一个不透明的标题栏。标题栏区域变为透明的。
- 你的 Web 内容可以延伸并绘制到整个窗口，包括原来标题栏所在的区域。
- 你必须自己在 HTML/CSS 中创建一个假的“标题栏”元素（一个 div），并为其设置样式（背景色、按钮等）。
- 你必须自己实现这个自定义标题栏的拖拽功能来移动窗口（通常通过给自定义标题栏元素添加 data-tauri-drag-region 属性实现）。
- 窗口的控制按钮（交通灯按钮）仍然由系统绘制，它们会浮动在你的 Web 内容之上。你需要通过 CSS 为你的内容区域添加 padding 或 margin，以确保重要内容不会被这些按钮遮挡。

视觉效果：

```
+—[透明区域，系统按钮浮动在上面]———————————————+
|  [绿灯] [黄灯] [红灯]                          | <-- 系统按钮（浮动层）
|  +——————————————————————————————+             |
|  |  你自己用 HTML 做的标题栏    |             | <-- 你的 Web 内容（延伸到了顶部）
|  +——————————————————————————————+             |
|  |                                            |
|  |          你的 Web 应用内容区域             |
|  |                                            |
|  |                                            |
+————————————————————————————————————————————————
```

**TitleBarStyle::Overlay**

这是 Transparent 的一个变体，主要目的是为了解决 Transparent 模式下一个特定的痛点：系统窗口控制按钮的遮挡问题。

行为：

- 与 Transparent 几乎完全相同：标题栏透明，Web 内容延伸到整个窗口，你需要自己实现自定义标题栏和拖拽区域。
- 关键区别：系统窗口控制按钮（交通灯按钮）不再是一个浮动在顶层的独立层。相反，它们被嵌入到了你的窗口内容中，成为窗口内容的一部分。
- 这意味着系统按钮会遮挡你的内容。你必须使用 CSS（如 padding-left 或 margin-left）主动为它们腾出空间，否则你的内容（比如一个导航栏）就会被按钮挡住。

视觉效果：

```
+————————————————————————————————————————————————
| [绿灯] [黄灯] [红灯]  <-- 这些按钮现在是你窗口内容的一部分，会和你写的元素重叠
|
|  +——————————————————————————————+
|  |  你自己用 HTML 做的标题栏    |  <-- 你的标题栏需要从右边开始，给左边按钮留出空间
|  +——————————————————————————————+
|  |                                            |
|  |          你的 Web 应用内容区域             |
|  |                                            |
|  |                                            |
+————————————————————————————————————————————————
```

### sea-orm 集成

```shell
cargo install sea-orm-cli
```

2. 创建Migration

在`src-tauri`目录下，执行

```shell
sea-orm-cli migrate init
```
